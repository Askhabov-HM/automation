# Versioning

Project versioning must follow Semantic Versioning:

`MAJOR.MINOR.PATCH`

## Version bump rules

- `fix` -> `PATCH`
- `feat` -> `MINOR`
- any breaking change -> `MAJOR`

## Rules

- Do not modify an already released version.
- Any released change must produce a new version.
- Public API breaking change requires major version bump.
- Backward-compatible feature requires minor version bump.
- Backward-compatible bug fix requires patch version bump.
